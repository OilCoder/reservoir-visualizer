## 16:23 — Entorno Docker, Phase 1 completa, y mapa de litologías con sustento institucional

### Technical changes
- `Dockerfile` + `docker-compose.yml` — entorno aislado Python 3.11-slim; volúmenes para hot-reload de `src/`, `data/`, `tests/`, `app.py`
- `requirements.txt` — dependencias pinned: lasio, numpy, pandas, scipy, plotly, streamlit, pyyaml, pytest
- `src/las/loader.py` — lee LAS 2.0 con lasio, reemplaza -999.25 por NaN, extrae metadata (well_name, field, api, lat/lon, elev, curves, raw_other)
- `src/las/formation_parser.py` — parsea CSV en sección ~Other, ordena por TOP, infiere BASE faltante del siguiente TOP
- `src/las/lithology_map.py` — FORMATION_LITHOLOGY (KGS Bulletin 189) + LITHOLOGY_COLORS (USGS TM 11-B1); resolución con fallback fuzzy
- `tests/` — 28 tests, todos verdes corriendo dentro de Docker
- `docs/02_lithology_map.md` — documentación con tabla de colores, cobertura de formaciones y referencias citables

### Design decisions
- Docker en vez de venv en WSL — aislamiento total entre proyectos, reproducibilidad garantizada para deploy en HF Spaces
- `--server.fileWatcherType=poll` en CMD del Dockerfile — sin esto Streamlit no detecta cambios en WSL2
- Plotly 3D confirmado (vs PyVista) — sin dependencia VTK, integración nativa Streamlit, funciona en HF Spaces headless
- Litología por nombre de formación (KGS Bulletin 189) en vez de clasificación por curvas — más simple para Phase 2, El Dorado está bien documentado; fallback por GR queda para mejora futura
- Colores USGS TM 11-B1 en vez de FGDC-STD-013-2006 — mismo estándar visual, CMYK de FGDC no está disponible como tabla hex; USGS TM 11-B1 sí es directamente citable y usable
- Fuzzy match en `get_formation_lithology` — los LAS de El Dorado tienen variantes de nombres (Heebner vs Heebner Shale, Stark vs Stark Shale); el match evita falsos Unknown

### Pending
- [ ] Implementar `src/render/single_well.py` — bloques 3D Plotly coloreados por litología
- [ ] Script debug `debug/dbg_single_well_render.py` — validar figura con datos reales de El Dorado

### Learnings
- Los LAS de El Dorado no siempre tienen las filas de ~Other ordenadas por profundidad (Barnhill_A_32 empieza con Stark antes que Oread) — el sort por TOP es obligatorio antes de inferir bases
- Algunos archivos tienen ~Other vacío (Batman_A_23, Bloir_A_33) — el parser debe retornar [] sin error, no lanzar excepción
- La cadena vacía `""` es substring de cualquier string en Python — guardar contra empty string antes del fuzzy match fue un bug real detectado por los tests

## 17:45 — Phase 4 completa: UI Assembly

### Technical changes
- `data/config.yaml` — configuración centralizada: `app.max_demo_wells=6`, `render.single_well_width_ft=200`, `render.multi_well_grid_n=40`, `render.figure_height_px=650`
- `src/ui/sidebar.py` — `render_sidebar()`: file uploader (LAS múltiples), botón demo El Dorado con `@st.cache_data`, radio view mode (single/multi), selectbox de pozo, multiselect de formaciones
- `src/ui/viewer.py` — `render_viewer()`: fila de 4 métricas (wells, formations, 3D surfaces, max depth) + `st.plotly_chart` con altura configurable
- `app.py` — entry point Streamlit: `set_page_config(layout="wide")`, inicialización de `session_state.wells_data`, despacho a `build_single_well_figure` o `build_multi_well_figure` según view_mode, empty state con mensaje de bienvenida

### Design decisions
- `app.py` lee `wells_data` de `st.session_state` después de llamar `render_sidebar` — la sidebar escribe el estado y el main loop lo lee; sin callbacks ni rerun explícito
- Filtrado de formaciones en `app.py`, no en `sidebar.py` — el sidebar solo retorna los nombres seleccionados, el main loop filtra la lista antes de pasar a los renderers
- `main()` retorna temprano en empty state — evita llamar a los renderers con lista vacía, sin try/except adicional
- `str(well_name)` al pasar a `build_single_well_figure` — el campo `well_name` puede ser entero en algunos LAS de Kansas (bug detectado previamente en debug)
- `app.py` era un directorio vacío (artefacto del volumen Docker) — eliminado con `rmdir` antes de crear el archivo
